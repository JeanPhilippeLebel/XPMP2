13 /*
14  * Theory of Operation
15  *
16  * Plug-ins can create menus in the menu bar of X-Plane. This is done by
17  * creating a menu and then creating items. Menus are referred to by an
18  * opaque ID. Items are referred to by (zero-based) index number.
19  *
20  * Menus are "sandboxed" between plugins---no plugin can access the menus of
21  * any other plugin. Furthermore, all menu indices are relative to your
22  * plugin's menus only; if your plugin creates two sub-menus in the Plugins
23  * menu at different times, it doesn't matter how many other plugins also
24  * create sub-menus of Plugins in the intervening time: your sub-menus will be
25  * given menu indices 0 and 1. (The SDK does some work in the back-end to
26  * filter out menus that are irrelevant to your plugin in order to deliver
27  * this consistency for each plugin.)
28  *
29  * When you create a menu item, you specify how we should handle clicks on
30  * that menu item. You can either have the XPLM trigger a callback (the
31  * XPLMMenuHandler_f associated with the menu that contains the item), or you
32  * can simply have a command be triggered (with no associated call to your
33  * menu handler). The advantage of the latter method is that X-Plane will
34  * display any keyboard shortcuts associated with the command. (In contrast,
35  * there are no keyboard shortcuts associated with menu handler callbacks with
36  * specific parameters.)
37  *
38  */
