254 /***************************************************************************
255  * VIRTUAL KEY CODES
256  ***************************************************************************/
257 /*
258  * These are cross-platform defines for every distinct keyboard press on the
259  * computer. Every physical key on the keyboard has a virtual key code. So
260  * the "two" key on the top row of the main keyboard has a different code
261  * from the "two" key on the numeric key pad. But the 'w' and 'W' character
262  * are indistinguishable by virtual key code because they are the same
263  * physical key (one with and one without the shift key).
264  *
265  * Use virtual key codes to detect keystrokes that do not have ASCII
266  * equivalents, allow the user to map the numeric keypad separately from the
267  * main keyboard, and detect control key and other modifier-key combinations
268  * that generate ASCII control key sequences (many of which are not available
269  * directly via character keys in the SDK).
270  *
271  * To assign virtual key codes we started with the Microsoft set but made some
272  * additions and changes. A few differences:
273  *
274  * 1. Modifier keys are not available as virtual key codes. You cannot get
275  * distinct modifier press and release messages. Please do not try to use
276  * modifier keys as regular keys; doing so will almost certainly interfere
277  * with users' abilities to use the native x-plane key bindings.
278  *
279  * 2. Some keys that do not exist on both Mac and PC keyboards are removed.
280  *
281  * 3. Do not assume that the values of these keystrokes are interchangeable
282  * with MS v-keys.
283  *
284  */
